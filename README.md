# RSケーニーズ 記録アーカイブ LAB

日本水泳連盟の公開結果を継続取得し、Firebaseへ保存してから表示する実験版PWAです。

## 実験版で変更した点

- 開催前はRSケーニーズの出場予定が取得できた大会だけを表示
- 開催前大会を押すと出場予定者を性別・年齢・種目で絞り込み、自己ベストを確認可能
- 学年の代わりに選手詳細で生年月日を登録し、満年齢を自動計算
- 生年月日から計算した年齢・性別・自己ベストを日本水泳連盟の2026年度資格級表と自動照合
- Firestoreの保存内容を最初に表示
- Firestoreの変更をリアルタイム反映
- アプリ起動時と起動中6時間ごとにバックグラウンド同期
- 手動更新ボタンを廃止
- 新規取得が空または失敗しても過去記録を削除しない
- 取得済み記録を安定IDで重複排除
- PWA画面キャッシュは使用しない

## データの流れ

```text
Results of Japan Swimming 公式API
  -> /api/swim-results
  -> RSケーニーズの結果と出場予定だけを整形
  -> Firestoreへ追記保存
  -> アプリへリアルタイム反映
```

資格級は `/api/qualification` が日本水泳連盟の公式表を取得し、端末に1日キャッシュして判定します。生年月日未設定の選手は「級未判定」と表示されます。

Firebaseの準備は `FIREBASE_SETUP.md` の順番で進めてください。

現在のFirebaseルールは、認証なしで誰でも閲覧・書き込みできる設定です。

## 現在の取得アダプター

`api/swim-results.js` は `result.swim.or.jp` の公開APIから所属名「ケーニーズ」を検索し、所属コード `22285` の選手だけを取得します。現在の登録選手46名を対象に、種目・長水路/短水路ごとの過去12か月の大会記録を取得し、アプリ形式へ変換します。

取得済み記録はアプリ起動時にFirestoreへ追記保存されます。公式サイト側から古い記録が消えたり、一時的に取得できなかった場合も、Firestoreに保存済みの記録は削除しません。
